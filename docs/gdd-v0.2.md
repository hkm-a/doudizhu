## D. 牌型规则（反向文档化自 rule.py）

> svzdev 的牌型规则通过 252K 查表文件 rule.json 加载，rule.py 负责识别 + 比较。
> 本章是**设计契约**——告诉玩家"什么牌能压什么牌"。

### D.1 牌面表示

- **数字表示**：1..52 = 4 花色 × 13 牌（A/2/3/.../K），53 = 小王，54 = 大王
- **字符表示**：`34567890JQKA2wW`（按从小到大排序）
- **王**：53 → `w`（小王），54 → `W`（大王）

### D.2 牌型清单（v0.1 rule.py CARD_TYPES）

| 牌型 | 张数 | 例子 | 设计意图 |
|------|------|------|----------|
| `single` 单张 | 1 | 5 | 最小操作单位 |
| `pair` 对子 | 2 | 55 | 紧邻策略 |
| `trio` 三张 | 3 | 555 | 中等压力 |
| `trio_single` 三带一 | 4 | 5556 | 灵活配牌 |
| `trio_pair` 三带二 | 5 | 55566 | 灵活配牌 |
| `seq_single5..12` 顺子 | 5–12 | 34567 | 核心操作 |
| `seq_pair3..10` 连对 | 6–20 | 334455 | 顺子升级版 |
| `seq_trio2..6` 飞机 | 6–18 | 333444 | 高级压力 |
| `seq_trio_single2..5` 飞机带单 | 8–20 | 33344456 | 高级压力 + 灵活 |
| `seq_trio_pair2..4` 飞机带对 | 10–20 | 3334445566 | 高级压力 + 灵活 |
| `bomb` 炸弹 | 4 | 5555 | 强压制 |
| `rocket` 王炸 | 2 | wW | 顶级压制 |
| `bomb_single` 四带二单 | 6 | 55556x | 变种炸弹 |
| `bomb_pair` 四带二对 | 8 | 555566xx | 变种炸弹 |

### D.3 牌型压制规则（自 compare_pokers）

- **同牌型比大小**：`a_card_value - b_card_value`
- **不同牌型**：
  - 火箭（`rocket`，value=22000）压一切
  - 炸弹（`bomb`，value=20000+rank）压所有非炸弹/火箭
  - 其他牌型必须同牌型 + 大过才压
- **大过约束**：
  - 单张：A > 2 > ... > 3
  - 对子/三张/炸弹：同单张
  - 顺子：连续长度 + 首牌大小
  - 连对/飞机：连续组数 + 首组大小

### D.4 牌型设计契约（玩家视角）

- **火箭是顶级**：比炸弹大、比所有牌型大
- **炸弹是越级牌**：能压任何非炸弹非火箭
- **炸弹比炸弹**：按牌面大小（5555 > 4444）
- **同牌型必须同长度**：5556 不能压 555566（4 张 vs 5 张）
- **2 和王不能进顺子/连对/飞机**：[待测试] 当前 rule.py 实现

### D.5 调参杠杆（牌型相关）

- 牌型查表 rule.json **禁止硬编码修改**（GDD v0.1 B 章节已声明）
- 牌型判定逻辑改 rule.py 必须**双向测试** + 跑 AI 决策日志对照
- 新增牌型**必须新增 GDD 章节** + 调参表条目 + AI 性格影响

## E. 计分（基础 + 加倍后）

### E.1 基础计分（v0.1 已实装）

- **底分（origin）**：叫分结果（0/1/2/3 分）
- **当前倍数（multiple）**：从 origin 累加
- **触发条件**（[待测试] svzdev 是否全部实装）：
  - 炸弹：multiple += 1（[待测试]）
  - 火箭：multiple += 2（[待测试]）
  - 春天：multiple *= 2（[待测试]）
  - 反春天：multiple *= 2（[待测试]）

### E.2 结算公式

```
赢家得分 = 底分 × 当前倍数 × 角色修正
```

| 角色 | 公式 |
|------|------|
| 地主赢 | +底分 × multiple × 2 |
| 农民赢 | +底分 × multiple × 1（两个农民各 +1 倍） |
| 地主输 | -底分 × multiple × 2 |
| 农民输 | -底分 × multiple × 1（每个农民 -1 倍） |

**注**：当前 svzdev 是否**完整实装**计分逻辑 [待测试]。需要查 `room.py` + `player.py` 的结算分支。

### E.3 加倍后计分（v0.2 新工作）

按 **C.4 加倍机制** 实施后：

```
最终倍数 = 基础 multiple × 加倍系数（每个加倍的玩家 +1×）
```

| 场景 | multiple | 加倍数 | 最终 |
|------|----------|--------|------|
| 1 个农民加倍 | 1 | +1 | 2× |
| 2 个农民都加倍 | 1 | +2 | 3× |
| 1 个农民 + 地主都加倍 | 1 | +2 | 3× |
| 全部加倍 | 1 | +3 | 4× |
| 无加倍 | 1 | 0 | 1× |

[待测试] 加倍系数是否为 +1 还是 +2（**设计杠杆**——影响博弈张力）

### E.4 计分可触达杠杆（支柱 5 落地）

| 杠杆 | 当前 | v0.2 起步 | v0.3 候选 |
|------|------|----------|----------|
| 叫分倍率（origin） | 0–3 | 0–3 | 0–5（[待测试]） |
| 炸弹倍率 | +1× [待测试] | +1× | +2× [待测试] |
| 火箭倍率 | +2× [待测试] | +2× | +3× [待测试] |
| 春天倍率 | ×2 [待测试] | ×2 | ×3 [待测试] |
| 反春天倍率 | ×2 [待测试] | ×2 | ×3 [待测试] |
| 加倍系数 | 未实装 | +1× | +2× [待测试] |
| 顶倍上限 | 无 [待测试] | 16× [待测试] | 32× [待测试] |

## F. AI 行为契约（性格注入层）

> 支柱 3 "AI 有脾气" 的设计落地。
> 规则 AI 当前是 **单一性格**（`RuleBasedPolicy`），DouZero AI 是 **训练数据决定**。
> 本章设计**性格注入层**，让玩家在房间配置里选择 AI 性格。

### F.1 性格模式清单

| 性格 | 描述 | 调参影响 | 设计意图 |
|------|------|----------|----------|
| `conservative` 保守 | 抢地主阈值高、跟牌谨慎、保留大牌 | 抢地主阈值 ≥5 张大牌、保留火箭、让牌阈值提高 | 新手引导专用 |
| `balanced` 平衡 | svzdev 原启发式默认 | 当前实现 | 默认 |
| `aggressive` 激进 | 抢地主阈值低、敢炸、敢春天 | 抢地主阈值 ≥3 张大牌、敢出炸弹、敢叫分 | 高手玩家 |
| `trickster` 老六 | 会扣牌、做牌、算炸弹、让牌给队友 | 抢地主策略强化扣牌、跟牌"反向思考" | 风格化对手 |
| `erratic` 失误 | 偶尔不按策略、模拟人类失误 | 5% 概率随机出 [待测试] | 风格化对手 |

### F.2 性格注入层 API 草案

```python
class PersonalityMode(Enum):
    CONSERVATIVE = 'conservative'
    BALANCED = 'balanced'
    AGGRESSIVE = 'aggressive'
    TRICKSTER = 'trickster'
    ERRATIC = 'erratic'

@dataclass
class PersonalityConfig:
    mode: PersonalityMode
    rob_threshold: int           # 抢地主需要几张大牌
    follow_strategy: str         # 'strict' | 'relaxed' | 'reverse'
    bomb_threshold: int          # 何时敢出炸弹（[待测试]）
    rocket_hold_threshold: int   # 火箭保留时对手剩几张
    let_ally_threshold: int      # 队友剩几张时让牌
    err_rate: float              # 失误率（仅 erratic）

def get_personality_config(mode: PersonalityMode) -> PersonalityConfig:
    """性格 → 调参配置 的查表。"""
    if mode == PersonalityMode.CONSERVATIVE:
        return PersonalityConfig(mode, rob_threshold=5, follow_strategy='strict',
                                 bomb_threshold=10, rocket_hold_threshold=15,
                                 let_ally_threshold=2, err_rate=0.0)
    # ... 其他性格
```

### F.3 性格选择入口

- **房间配置**：创建房间时 `personality` 字段（默认 `balanced`）
- **机器人列表**：玩家选择机器人时显示性格（[待设计] UI）
- **多机器人混搭**：一桌 2 机器人可以不同性格（[待设计]）
- **固定牌局回放**：`ai-replay-smoke.py` 接受 `personality` 参数

### F.4 性格 vs 策略

- **性格 ≠ 策略**：
  - 策略 = RuleBasedPolicy / DouZeroPolicy（技术实现）
  - 性格 = 调参组合（行为表现）
- **性格可叠加**：RuleBasedPolicy + Aggressive 性格 / DouZeroPolicy + Conservative 性格 都允许
- **性格不替换策略**：性格不改模型本身，只改决策阈值

### F.5 性格的"可读性"（支柱 3 落地）

按 SKILL.md "AI 有脾气" 的可测标准：
- 性格必须在 **AI 决策日志**里**显式记录**：`policy.py` 的 `decision_event` 增 `personality` 字段
- 玩家回放能复述"AI 在 X 时刻犯了 Y 错" → 失误来自 `erratic` 性格
- 玩家回放能复述"AI 风格偏 Z" → 来自性格标识

### F.6 性格实施路径

- **v0.2 起步**：
  - 创建 `server/ai/personality.py` 模块
  - 5 种性格配置 dataclass
  - `Policy.choose_rob / choose_shot` 接受 `personality` 参数
  - 决策日志增 `personality` 字段
- **v0.2 验证**：
  - 跑 5 局同牌局、5 种性格 → 决策日志对比
  - 验证每种性格的"可读"性
- **v0.3 候选**：
  - 玩家手动调性格参数
  - 性格随玩家历史自适应

## G. 加倍机制（v0.1 整章未实装 → v0.2 新工作）

> 支柱 5 "输赢由选择不由运气" 的**核心新功能**。
> 设计师 + 工程师协同工作。

### G.1 机制设计

- **触发时机**：地主确认后、地主第一次出牌**前**
- **可加倍方**：
  - 2 个农民：各 1 次选择权
  - 地主：1 次选择权（在农民选择后）
- **选择**：加倍 / 不加倍
- **限制**：
  - 每人 1 次选择权（不能再加）
  - 选择顺序：农民 1 → 农民 2 → 地主
  - 超时自动"不加倍"（不打断对局）
- **效果**：
  - 加倍的玩家：最终 multiple += 加倍系数（默认 1×）
  - 输赢按最终 multiple 计算（见 E.3）

### G.2 玩家幻想

- **加倍**："我赌这局我能赢 / 我要拖队友下水"
- **不加倍**："我牌力不够 / 不想冒险"
- **可读性**：其他玩家的加倍选择**公开可见**（[待设计] 是否隐藏）

### G.3 边界情况

- **农民 1 加倍 + 农民 2 不加倍**：multiple += 1
- **农民都加倍**：multiple += 2
- **农民都加倍 + 地主加倍**：multiple += 3
- **农民都不加倍 + 地主加倍**：multiple += 1（地主对自己加倍）
- **超时**：默认"不加倍"
- **离线 / 托管**：托管 AI 按 personality 决策

### G.4 AI 加倍策略（性格驱动）

| 性格 | 加倍倾向 | 原因 |
|------|----------|------|
| Conservative | 罕见加倍 | 不冒险 |
| Balanced | 看手牌强度 | 阈值 [待测试] |
| Aggressive | 频繁加倍 | 豪赌型 |
| Trickster | 心理战 | 反向思考 |
| Erratic | 5% 概率随机 | 失误模拟 |

**v0.2 实施**：`RuleBasedPolicy.choose_double(player, room, personality)` 方法
**v0.2 验证**：跑 100 局同牌局 5 性格 → 加倍分布统计

### G.5 加倍 UI（[待设计]）

按支柱 1 "3 秒读局"：
- HUD 加倍状态区："X 加倍 / Y 不加倍 / Z 待定"
- 加倍按钮**位置显眼**
- 加倍动画 + 音效（**新增**——v0.2 必做，缺失音频清单 1）

### G.6 加倍与新机制的协同

按支柱 5 "可触达机制数 ≥ 3"：
- 当前：叫分（1）
- v0.2：+ 加倍（2）
- v0.3 候选：+ 明牌 / 换牌 / 换庄（3+）

## I. 9 个缺失音效规格书

> 支柱 2 "每个决策有反馈" 的资产缺口。
> v0.2 必做：写规格书 + 占位文件 + 标记待真实音频替换。

### I.1 规格清单

| 编号 | 音效 | 时长 | 频率/调性 | 情绪/能量 | 触发动作 | 缺失/完成 |
|------|------|------|----------|----------|----------|-----------|
| S1 | 炸弹 | 1.0s | 200Hz → 100Hz 扫频 | 沉重冲击 | 4 张同号出牌 | ❌ → 占位 |
| S2 | 火箭 | 1.2s | 500Hz → 1500Hz → 200Hz | 升空 + 爆炸 | 王炸出牌 | ❌ → 占位 |
| S3 | 春天 | 0.8s | 800Hz 持续 + 200Hz 钟声 | 胜利宣告 | 地主一手清完 | ❌ → 占位 |
| S4 | 反春天 | 0.8s | 200Hz 钟声 + 100Hz 低音 | 反派胜利 | 农民各出一手 | ❌ → 占位 |
| S5 | 托管 | 0.6s | 400Hz → 600Hz 双音 | 中性提示 | 玩家进入托管 | ❌ → 占位 |
| S6 | 不出 | 0.4s | 300Hz 短音 + 衰减 | 快速反馈 | pass 动作 | ❌ → 占位 |
| S7 | 出牌 | 0.5s | 500Hz 短音 + 高频泛音 | 干脆 | 普通出牌 | ❌ → 占位 |
| S8 | 倒计时 | 0.3s × 3 | 600Hz 嘀嘀嘀 | 紧迫 | 超时前 3/2/1 秒 | ❌ → 占位 |
| S9 | 加倍 | 0.8s | 700Hz → 1000Hz 上升 | 赌注加码 | 加倍选择 | ❌ → 占位 |

### I.2 占位文件生成方案

- **生成方式**：Python + numpy 合成 sine wave + 简单包络
- **采样率**：44.1kHz / 16-bit / mono（与现有 mp3 格式匹配）
- **输出格式**：MP3（如 lameenc 可用）或 WAV（兜底）
- **命名规范**：`{name}_placeholder.mp3`（清晰标记占位）
- **不替换真实音频**：仅占位，**待真实音频资产替换**

### I.3 真实音频资产建议

- **资产来源**：[Freesound.org](https://freesound.org) / [Pixabay Audio](https://pixabay.com/audio/)
- **版权**：CC0 / CC-BY（标注作者）
- **替代方案**：用 AI 音乐生成（[Suno](https://suno.com) / [HeartMuLa](https://github.com/...)）— 需用户授权
