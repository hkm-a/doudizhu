## A. 核心循环三层

### A.1 即时体验（0–30 秒）—— 抢地主

- **行为**：3 名玩家依次决定"叫 / 不叫"当前底分
- **反馈**：
  - 听到前序玩家叫分（4 套男声 / 4 套女声按性别播放）
  - 看到自己手牌 + 抢分按钮
  - 牌桌中央显示当前"底分"从 0 → 1 → 2 → 3
- **玩家此刻的感受**：紧张 / 算牌 / 抢 / 装怂
- **玩家可做决策**：
  - 叫 1/2/3 分
  - 不叫（pass）
  - 抢已叫的分数（"加叫"）

### A.2 单局循环（5–30 分钟）—— 出牌

- **行为**：地主先出，农民轮流出牌 / 不出
- **反馈**：
  - 听到叫分音效 / 发牌音效 / 胜负音效
  - 看到残影区（最近出牌历史）
  - 牌面按 3:4 比例绘制（`server/static/js/generator.mjs`）
- **玩家可做决策**：
  - 出牌（按牌型规则：单 / 对 / 三 / 三带 / 顺子 / 连对 / 飞机 / 炸弹 / 火箭）
  - 不出（pass）
  - 拖时间到托管（超时自动托管，详见 C.6）

### A.3 长期循环（数小时–数周）—— 留待设计

**当前状态：[待设计]** —— 项目当前没有元进度 / 赛季 / 段位 / 每日牌局 / 好友房 等长期循环。

按支柱 5 "输赢由选择不由运气"和支柱 3 "AI 有脾气"，**v0.2 GDD 起步**应包括：
- 段位 / 积分体系（基于倍数累计）
- 每日牌局数（防沉迷 + 留存钩子）
- 赛季 / 排行榜
- 好友房 / 自定义桌

**当前实测行为**：服务端 `GAME_OVER` 后 5 秒机器人自动 ready（`simple.py:53`），**单局间无中断**——这是天然的"再来一局"钩子，但需要配套玩家行为日志（v0.1 后续章节）来验证玩家是否跟随。

## C. 机制规格书

> 按 SKILL.md 机制规格书模板：目的 / 玩家幻想 / 输入 / 输出 / 成功判定 / 失败状态 / 边界情况 / 调节杠杆 / 依赖

### C.1 叫地主（抢地主）

- **目的**：决定谁拿底牌 + 当地主（高风险高回报）
- **玩家幻想**：我手牌 + 底牌 = 一手好牌，敢抢敢赢
- **输入**：
  - 玩家点击"叫 X 分"按钮
  - 或点击"不叫"
  - 上一手已叫分则可"加叫"到更高分
- **输出**：
  - 房间底分 origin + 1（按玩家叫分数）
  - 服务器广播 `RSP_CALL_SCORE`
  - 牌桌播放对应 m_score_X.mp3 / f_score_X.mp3
- **成功判定**：
  - 3 人中有人叫 3 分 → 立即成为地主
  - 3 人叫分轮完后，叫分最高者成为地主
- **失败状态**：
  - 3 人都"不叫" → 流局，本局无效
  - 玩家叫分超时 → 自动 pass（[待测试] 是否有超时兜底）
- **边界情况**：
  - 第 2 轮"加叫"是否可加到自己前轮的分数？（[待测试] svzdev 规则可能不允许）
  - 加叫到 3 分直接成地主？（[待测试]）
- **调节杠杆**：
  - 底分上限
  - AI 抢地主策略（手牌含几张大牌就抢）
- **依赖**：发牌 / 手牌可见性 / 底牌机制

### C.2 出牌

- **目的**：通过牌型规则压制对手 / 队友配合
- **玩家幻想**：甩出"一手大牌"的爽感 / 算到对手剩什么的智识满足
- **输入**：
  - 玩家点击手牌（单选/多选）+ 点击"出牌"
  - 或点击"提示"（自动选最小可压牌）
  - 或点击"不出"
- **输出**：
  - 牌移动到残影区
  - 服务器校验牌型
  - 当前倍数 multiple 累加（如有炸弹 / 火箭）
  - 广播 `RSP_SHOT_POKER`
- **成功判定**：
  - 牌型合法 + 大过上一手 → 出牌成功，轮次切换到下一玩家
  - 牌型非法 → 服务器拒绝
  - 牌型合法但压不过 → 服务器拒绝（规则上也可出但是会被打回）
- **失败状态**：
  - 超时 → 触发托管（详见 C.6）
  - 选错牌型（不构成合法牌型）→ 服务器校验失败
  - 残影区无牌 + 没出牌 → 自由出牌（first move）
- **边界情况**：
  - 玩家选错多张牌（如三带一多选）→ 前端 UI 需标灰禁用
  - 火箭在手 + 对手剩 10+ 张 → 保留不出（AI 已实装，玩家 UI 是否提示？[待设计]）
  - 队友剩 4 张 + 我手多 4 张 → 队友优先出（让牌）→ UI 是否提示？[待设计]
- **调节杠杆**：
  - 牌型查表 rule.json（252K 查表，**禁止硬编码修改**）
  - 炸弹/火箭倍率
  - 出牌超时
- **依赖**：牌型规则 / 残影区 UI / 倍数系统

### C.3 不出（pass）

- **目的**：放弃本轮出牌，让下一玩家压牌
- **玩家幻想**：算到队友能压 / 自己压不过 / 留牌走长牌
- **输入**：玩家点击"不出"按钮
- **输出**：服务器广播"不出"，本轮结束逻辑推进
- **成功判定**：服务器记录"不出"事件，进入下一玩家决策
- **失败状态**：
  - 是"轮首玩家"且未出牌 → **不允许不出**（前端要禁用按钮）
  - 自由出牌轮（残影区无牌）→ 不允许不出
- **边界情况**：
  - 连续 N 轮都"不出" → 自由出牌轮（下一个出牌者重新自由出）
  - 玩家故意"不出"以保留大牌 → 这就是策略，**允许**
- **调节杠杆**：无
- **依赖**：轮次状态机

### C.4 加倍

**当前状态：[未实装]** —— 反向文档化发现 svzdev 原项目**没有**加倍机制。

按支柱 5 "输赢由选择不由运气"，**v0.2 GDD 必须补**：
- **目的**：放大决策权重，让"赢 1 张"由玩家叫出来
- **机制草案**：
  - 出牌前可对当前轮次"加倍"（如农民对地主加倍，倍数×2）
  - 倍数上限 [待测试]
  - 是否允许"反加倍"（[待设计]）
  - 火箭炸弹触发自动加倍 vs 玩家主动选择加倍
- **可触达机制数**：当前只有 1（叫分），加上加倍后达到 2

### C.5 明牌 / 底牌

- **底牌**：3 张，地主专属
- **明牌**：[待测试] 当前是否实装

### C.6 托管

- **目的**：避免玩家掉线/超时卡住对局
- **触发条件**：[待测试] 推测 N 次超时自动进入
- **行为**：自动按 AI 策略出牌 + 自动 ready
- **玩家体验影响**：
  - 正面：避免卡局
  - 负面：被托管玩家**失去博弈张力**（按支柱 3，玩家需要"有脾气的对手"，托管 AI 不一定满足）
- **调节杠杆**：
  - 触发阈值
  - 托管 AI 是否跟非托管 AI 一样（"有脾气"）
  - 托管结束条件（玩家回来后是否继续托管）

## 跟 AI 决策日志的对应

AI 决策日志（`policy.py` 已实装）记录：
- `policy`（rule / douzero）
- `action`（rob / shot）
- `player` + `room` 上下文
- `decision` 结果
- `reason`（如 `ally_has_few_cards_left`、`hold_rocket_while_opponent_has_many_cards`）
- `douzero_action`（rank 级别动作）

**v0.1 GDD 的 AI 契约**（支柱 3 "AI 有脾气" 落地）：
- 规则 AI 的"脾气"由调参表控制（抢地主阈值、跟牌策略阈值、火箭保留阈值）
- DouZero AI 的"脾气"目前由训练数据决定，[待设计] 性格注入层

**v0.2 GDD 必须补**：
- AI 性格模式清单（激进 / 保守 / 读牌 / 失误）
- AI 性格选择入口（房间配置或机器人选择）
- AI 性格注入层 API 草案
