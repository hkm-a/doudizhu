## 设计支柱

### 支柱 1："3 秒读局"（UI 主导）

**承诺**：玩家扫一眼牌桌，3 秒内知道 —— 我的牌 / 对手剩几张 / 该谁出 / 底牌 / 当前轮次。

**为什么**：信息密度不足会让玩家每步都问"现在啥情况"，博弈张力瞬间塌方。牌桌 UI 的首要职责是**信息透明**。

**可测的成功标准**：
- 玩家首次开桌后 3 秒内能正确回答"现在谁出牌"
- 残影区（最近 8 手已出牌）始终可见
- 计时器常驻可读，无须点击展开
- 牌桌不需要任何文字说明也能运行

**当前差距**：[待测试] Phaser 牌桌未做信息密度审计；docs/doudizhu-high-fidelity-ui.html 是高保真设计稿，未确认合入主线。

### 支柱 2："每个决策有反馈"（UI + 博弈）

**承诺**：叫分 / 抢地主 / 出牌 / 不出 / 加倍 —— **每一个**玩家选择都有 200ms 内的视觉 + 听觉反馈。

**为什么**：玩家**感觉到自己刚刚选了什么**，不是默默生效。反馈延迟 = 决策权重感知的直接损失。

**可测的成功标准**：
- 每个动作 → 反馈延迟 < 200ms（首字节到达播放设备）
- 玩家事后能准确回忆"我刚才按了什么 / 触发了什么效果"
- 反馈三通道一致：视觉（牌移动 / 高亮）+ 听觉（音效）+ 触觉（桌面端可选）
- 反馈**区分成功与失败**：大过牌 ≠ 被压牌 的音效

**当前差距**：[待测试] 已有 m_score_0..3.mp3 + f_score_0..3.mp3（4 套叫分音效），未审计每个动作的反馈链路是否闭环。炸弹、春天、托管触发等**特殊动作**的反馈未确认。

### 支柱 3："AI 有脾气"（博弈 主导）

**承诺**：规则 AI 和 DouZero AI 都有**可读的性格** —— 激进 / 保守 / 会读牌 / 偶尔失误。不是"完美对手"，是**有风格的可读对手**。

**为什么**：可读对手 → 玩家能预判 → 输赢有"差点能赢"的回放价值。完美 AI 不可读，玩家输了只觉得"系统判我输"，无法转成"下次我 X 一下可能赢"。

**可测的成功标准**：
- AI 决策日志可识别性格模式（已具备 scripts/ai-decision-summary.py 基础）
- 玩家回放对局能复述"AI 在 X 时刻犯了 Y 错"或"AI 风格偏 Z"
- 不同 AI 策略可**显式选择**（规则 AI 难 / DouZero 难 / DouZero 简单 / "老六"风格）
- "老六"性格：会扣牌、会做牌、会算炸弹；激进性格：敢抢、敢炸、敢春天

**当前差距**：[待测试] RuleBasedPolicy 是否"可读"、是否需要性格注入层；DouZeroPolicy 推理结果的"性格化包装"是否需要额外策略层。

### 支柱 4："桌面端一等公民"（UI + 工程）

**承诺**：不是 web 套 Tauri 壳 —— **键盘快捷键 + 原生菜单 + 高分屏适配 + Win/Mac/Linux 一致体验**都要有。玩家分不出"这是 Tauri 做的"和"这是原生做的"。

**为什么**：桌面玩家对原生感的敏感度比移动端玩家高一个量级。任何"这是个网页"的视觉/交互/性能信号都会**瞬间破功**。

**可测的成功标准**：
- 高分屏（2x/3x）下视觉无模糊、无锯齿、无元素错位
- 核心操作（出牌、叫分、不出、托管）都有键盘快捷键
- 跨平台安装包在 Win/Mac/Linux 都能"下载即用"（数据库初始化已内化）
- 启动到可玩 < 10 秒（无网络依赖）

**当前差距**：[待测试] src-tauri/ 是否已配置高分屏与原生菜单资源；键盘快捷键覆盖率 0%；1.0 路线图"桌面端内置运行时依赖 / 数据库生命周期 / 三平台发布矩阵"未完成。

### 支柱 5："输赢由选择不由运气"（博弈 主导）

**承诺**：通过叫分倍率、加倍、明牌、底牌机制**放大决策权重**。"赢 1 张" 和 "输 1 张" 是玩家**自己叫出来的**而不是发出来的。

**为什么**：博弈张力 > 手牌运气。差 1 张没出 / 多 1 张没扣 都要让玩家回想起"我刚才那手应该 X"。

**可测的成功标准**：
- "差 1 张没出" 的对局回放率 > 30%（证明博弈张力存在）
- 玩家回放能复述"我那个选择导致..."
- 牌型规则可调杠杆（炸弹倍率、春天判断、托管时长）显式文档化
- 至少 3 个"放大决策"机制在产品中可触达（不只叫分一种）

**当前差距**：[待测试] svzdev 原版规则的"可调杠杆"清单；加倍机制、明牌机制、底牌换牌机制 当前是否实装。
