审核通过项

  • 首屏保留真实牌桌，不改动 Phaser 游戏逻辑。
  • 登录页、牌桌页共享同一套视觉 token，避免拼贴感。
  • 按 frontend-design 技能要求替换泛用字体栈，增加更有中文游戏大厅气质的标题字体。
  • 用非可见 data-ui-theme="happy-doudizhu" 标记设计目标，便于 E2E 验收。
  • 中心牌桌、两侧状态栏、底部指标符合斗地主同类项目常见结构。
  • 按钮、输入框、状态徽章、进度条和风险态均有明确层级。
  • 增加 1180px、760px、520px 响应式布局，移动端降为单列。
  • 生产构建已通过：npm --prefix client run build。

主要改动

  • 新增全局 UI token：桌毡绿、骨白、金币黄、大红、青蓝。
  • 重塑登录页为欢乐斗地主式大厅入口，房间选择像场次卡片。
  • 重塑游戏页为桌面 HUD，牌桌置中并增加椭圆桌面和金色包边暗示。
  • 按钮、房间卡、状态条和结算区统一为红金游戏质感。
  • 优化座位、倍数、计时、行动建议、快捷语和结算区域的视觉状态。
  • 保留原 className 和 DOM 结构，降低测试与业务逻辑影响。

后续复核风险

  • 当前已做构建验证，但还未在真实后端牌局中完整点测一局。
  • CSS 不再依赖旧版融合方向，后续截图需按欢乐斗地主参考重新评审。
  • Phaser 画布内容仍由游戏场景绘制，若场景内部颜色很旧，可继续统一牌桌素材。
