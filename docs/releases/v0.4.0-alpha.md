# v0.4.0-alpha — Contributor-Ready Refactor

**Release date:** 2026-06-03

Changes since [v0.2.0-alpha](v0.2.0-alpha.md).

## Changes

### Architecture
- Extracted pure game engine (`server/game/`) — eliminated Room↔GameRound duplication
- Separated state machine, protocol, rules, and persistence boundaries
- Refactored `rule.py` from singleton `rule` object to `Rule` class with dependency-injected rule table

### AI Integration
- CI nightly job (`checkpoint-nightly.yml`) downloads real DouZero checkpoints and runs fixed-replay verification
- AI decision logs + summary report scripts (`npm run ai:summary`, `npm run ai:report`)
- Personality layer (5 personalities: conservative→aggressive) integrated into robot play

### Frontend
- Phaser game table implements full call-score → double → play → game-over flow
- Onboarding hints system for new players
- Segment badge generation
- E2E tests: full game flow mock + UI theme validation (`npm run test:e2e`, 4 tests)

### Backend Testing (560 tests)
- Room: 78 test methods (87 with subTests)
- Player API: 103 test methods
- GameRound: 80 test methods
- Card rules: 63 test methods
- Timer: 13 test methods
- RobotPlayer: 22 test methods
- PlayerEventLogger: 21 test methods
- Views (parse_bool, decode_message, save_player_points): 16 test methods

### License & Governance
- `LICENSE.md` — transparent license status documentation
- `NOTICE.md` — comprehensive dependency license audit (Python/JS/Rust)
- Asset audit: inherited vs. newly authored assets clearly listed
- Code of Conduct, Security policy, Contributing guide, Issue/PR templates
