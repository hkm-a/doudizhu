# v0.2.0-alpha Release Notes

This is a draft release note for the first installable Linux alpha package. It is intentionally marked alpha: the desktop shell can launch the local backend, but the app still depends on a local MySQL service and system Python dependencies.

## Highlights

- Tauri desktop shell with startup page, retry action and backend health navigation.
- Backend preflight checks for required Python imports and MySQL TCP reachability.
- React login cleanup aligned with the backend `name` login payload.
- Rule-based AI fallback plus DouZero adapter boundaries, card mapping, `InfoSet` conversion and fixed-deal replay smoke tests.
- AI decision JSONL logging and summary tooling for comparing rule AI and DouZero behavior.
- Unified verification command covering backend, web, desktop, config, release metadata and whitespace checks.

## Build Artifact

The manual GitHub Actions workflow `Linux Alpha Package` builds and uploads the Debian artifact:

```text
src-tauri/target/release/bundle/deb/doudizhu_0.1.0_amd64.deb
```

The artifact is uploaded together with this release note for review before attaching it to a public GitHub Release.

The workflow also generates:

```text
src-tauri/target/release/bundle/deb/doudizhu_0.1.0_amd64.deb.manifest.json
```

The manifest records package metadata, byte size, SHA256 and required bundled files.

For upload, the workflow gathers the `.deb`, manifest, these notes and `SHA256SUMS` into:

```text
dist/linux-alpha/v0.2.0-alpha/
```

## Install Notes

Before launching the package, prepare:

- MySQL on `127.0.0.1:3306`.
- Database `ddz` initialized from `schema.sql`.
- User `ddz` / `ddz`, or matching `DOUDIZHU_DATABASE_URI`.
- System Python with `requirements.txt` dependencies installed.

The desktop startup page shows backend preflight results and keeps a retry button visible when startup fails.

## Known Limitations

- Python runtime, Python packages and MySQL are not bundled yet.
- The database initialization and migration lifecycle is still manual.
- The release currently targets Linux `.deb` only.
- The inherited `svzdev/doudizhu` code and assets still need a resolved project-wide license path.
- Real DouZero checkpoint replay is available as a strict command, but not yet scheduled in release infrastructure.

## Verification Before Publishing

Run these checks before promoting an artifact to a public release:

```bash
npm run verify
npm run tauri:build
npm run desktop:artifact-smoke
npm run desktop:artifact-manifest
npm run desktop:release-bundle
npm run desktop:release-check
```

If real DouZero checkpoints are available:

```bash
DOUZERO_ENABLED=1 DOUZERO_MODEL_DIR=/absolute/path/to/douzero/baselines/douzero_ADP \
  PYTHONPATH=server python3 scripts/ai-douzero-replay-smoke.py --require
```
