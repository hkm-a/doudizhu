## 引导目标

让玩家**在第一次会话中**：
1. 30 秒内**理解**自己能做什么（叫分 / 出牌 / 不出）
2. **第一次决策**是有意义的（不是引导动画代劳）
3. **第一次成功**是保证的（哪怕是流局）
4. 离开时**想再来一局**

## 玩家画像（首次）

- **经验**：可能玩过斗地主，也可能没玩过
- **设备**：桌面端（最常见）或 Web
- **情绪**：好奇 + 紧张（怕输 / 怕不会玩）
- **容错**：极低（一次挫败可能弃坑）
- **注意力**：高（"我看看到底是什么"），但持续时间短（30 秒内要看到东西）

## 引导时间线（首次会话 0–5 分钟）

### T+0s：进入对局（玩家刚发完牌）

- **场景**：17 张牌在手
- **玩家此刻感受**：懵（这是什么）
- **引导动作**：
  - **HUD 首屏**（3 秒内可见）：我的手牌 + 对手手牌数（"剩 X 张"）+ 当前轮次 + 倍数 + 计时
  - **不弹窗、不打断** —— 让玩家先看
- **对应对照**：支柱 1 "3 秒读局"
- **检查清单**：
  - [ ] 首屏信息密度 [待测试]（vs docs/doudizhu-high-fidelity-ui.html）
  - [ ] 不弹窗引导 [待测试]（vs 现有 frontend 是否有 on-boarding modal）

### T+5s：玩家看到抢分按钮

- **场景**：手牌下方有"叫 1/2/3 分" / "不叫"按钮
- **玩家此刻感受**：我能做什么
- **引导动作**：
  - 按钮**位置显眼**（不藏在菜单里）
  - 按钮**标签明确**（"叫 1 分" / "叫 2 分" / "叫 3 分" / "不叫"）
  - **音效已预加载**（点击就有 m/f_score_0..3.mp3 反馈）
- **对应对照**：支柱 2 "每个决策有反馈"
- **检查清单**：
  - [ ] 按钮位置 [待测试]
  - [ ] 音效预加载 [待测试]

### T+8s：玩家第一次决策（"不叫"）

- **场景**：玩家点了"不叫"
- **玩家此刻感受**：我点了什么
- **引导动作**：
  - 立即播放 `m_score_0.mp3` 或 `f_score_0.mp3`（按性别）—— **支柱 2 兑现**
  - 牌桌中央"底分"显示维持 0
  - 轮次切换到下一玩家（动画 [待测试]）
- **失败处理**：
  - 玩家没看清按钮 → 超时 → 自动 pass（[待测试] 当前是否实装）
- **首次成功保证**：
  - **"不叫"永远不算失败** —— 这是 GDD v0.1 C.1 失败状态里"3 人都'不叫'则流局，本局无效"的安全网
  - 玩家第一次决策**总是成功**
- **对应对照**：支柱 2、5

### T+15s：玩家看其他人叫分（被引导看 AI 决策）

- **场景**：其他玩家（机器人）依次叫分
- **玩家此刻感受**：看 AI 怎么玩
- **引导动作**：
  - 机器人叫分有音效 + 牌桌显示底分变化
  - **机器人 1.5 秒延迟**（`simple.py:71`）—— 给玩家时间"消化"决策
  - 玩家**不靠文字说明**就理解"叫分是干嘛"
- **对应对照**：支柱 3 "AI 有脾气"（让玩家在引导阶段就读 AI）

### T+30s：抢地主结束，玩家看到谁是地主

- **场景**：地主产生
- **玩家此刻感受**：我（或对手）是地主
- **引导动作**：
  - 底牌翻给地主（动画 [待测试]）
  - **地主身份切换视觉**（头像高亮 / 标记 [待测试]）
  - 倍数显示更新
- **首次成功保证**：
  - 无论玩家是不是地主，地主都有 3 张底牌 → **牌力提升 = "被偏爱"**
  - 农民被偏爱：分数 × 2（[待测试] svzdev 是否实装）
- **对应对照**：支柱 1、5

### T+45s：玩家第一次出牌

- **场景**：玩家是地主 / 农民轮到自己
- **玩家此刻感受**：我能出哪些牌
- **引导动作**：
  - 鼠标悬停手牌 → **高亮**（[待测试]）
  - 选中 → **拖动 + 高亮**（[待测试]）
  - 选错牌型（不构成合法牌型）→ **服务器校验失败** → UI 提示（[待测试]）
  - "提示"按钮 → 自动选最小可压牌（[待测试] 当前是否实装）
- **反馈延迟目标**（支柱 2）：
  - 选牌 → 高亮 < 50ms
  - 出牌 → 牌移动 < 200ms
  - 出牌 → 音效 < 200ms（**当前缺失**，违反支柱 2）
- **首次成功保证**：
  - 第一手出"单张最小牌"是 100% 合法
  - 即使玩家"瞎出"，服务器都接受
- **对应对照**：支柱 2、5

### T+60s：玩家第一次跟牌（"不出"）

- **场景**：队友/对手出了大牌，玩家压不过
- **玩家此刻感受**：我该不该出
- **引导动作**：
  - "不出"按钮明显（[待测试]）
  - 点击 → 服务器接受 → 轮次推进
  - **"不出"音效缺失**（GDD v0.1 G 章节已确认）—— 这是引导第二大缺口
- **对应对照**：支柱 2

### T+5min：首局结束

- **场景**：玩家赢了 / 输了 / 流局
- **玩家此刻感受**：意犹未尽 / 想再来一局
- **引导动作**：
  - **胜利**：播放 `end_win.mp3` + 显示结算
  - **失败**：播放 `end_lose.mp3` + 显示结算
  - **流局**：不播放胜负 → **当前没有流局专用音效**（[待测试] 是否需要）
  - **机器人 5 秒后自动 ready**（`simple.py:53`）—— 玩家看到"机器人又来一局"
  - **玩家看到"开始下一局"按钮**（[待测试] 是否实装）
- **钩子设计**（支柱 5）：
  - **机器人自动 ready** = 视觉钩子（"游戏还在继续"）
  - **结算动画**（[待测试]）= 心理钩子（"我刚才那个选择导致..."）
  - **连胜/连败提示**（[待设计]）= 长期钩子

## 引导的"不要做"清单（反模式）

按支柱对应，**新手引导中绝对不能做的事**：

- ❌ **弹窗教学**（"按 1 叫分"）—— 打断玩家"探索"机制（违反检查 4）
- ❌ **强制观看视频/动画**（"先看 30 秒规则"）—— 玩家注意力短，立刻弃坑
- ❌ **AI 难度强制选择**（"先选简单/普通/困难"）—— 玩家没有参照系
- ❌ **首次对局就触发加倍/明牌** —— 玩家还没掌握基础
- ❌ **首次失败就扣分/降段** —— 玩家还没建立段位感
- ❌ **首次胜利就大肆庆祝**（多重大招动画）—— 喧宾夺主
