# 玩家首次开桌体验链诊断 v0.1

> 玩家从"双击桌面图标"到"出第一手牌"的全链路诊断。
> 静态代码审计 + 反向推断；不真启动避免动本地环境。
> 所有痛点标 `[待测试]` 需实测验证。

## 版本

| 版本 | 日期 | 摘要 |
|------|------|------|
| v0.1 | 2026-06-02 | 首版。基于 app.py / backend-preflight.py / boot.mjs / net.mjs / policy.py 反向推断。 |

## 诊断范围

- **桌面端启动链**：双击 .desktop → 启动页 → 预检 → 自动拉起后端 → 进对局
- **Web 端启动链**（绕开桌面）：开浏览器 → IndexHandler → Phaser Boot → 房间 → 对局
- **进对局**：登录/匿名 → 加入/创建房间 → 3 人齐 → 发牌 → 抢地主 → 出第一手牌

| 出牌音效 | 听觉 | **缺失** | ❌ | 2 |

- **整个"出牌"动作链 50% 反馈缺失**

## 5. 跨链总评

### 5.1 支柱违反密度（按本次诊断）

| 支柱 | 违反次数 | 关键环节 |
|------|----------|----------|
| 1. 3 秒读局 | 6 | 启动页 → 房间 → 牌桌首屏 → 选牌高亮 |
| 2. 每个决策有反馈 | 8 | 启动页预检 → 抢地主 → 地主切换 → 出牌音效 → AI 跟牌音效 |
| 3. AI 有脾气 | 1 | 托管 AI 性格缺失 |
| 4. 桌面端一等公民 | 8 | 启动页原生感 → 高分屏 → 键盘快捷键 → 启动到可玩时长 |
| 5. 输赢由选择不由运气 | 0 | 本次诊断范围内**无直接违反**（加倍机制缺位是 v0.2 工作） |

### 5.2 启动到出第一手牌的最严重缺口

按"玩家可感知度"排序：

1. **🔥 47% 音效完整度**（GDD v0.1 G 章节）—— 玩家**进对局后**听不到一半动作
2. **🔥 启动页 UI 视觉未审** —— 桌面端"原生感"未知
3. **🔥 牌桌首屏信息密度未审** —— 支柱 1 关键
4. **⚠️ 出牌反馈链 50% 缺失** —— 玩家出手的"爽感"没起来
5. **⚠️ AI 跟牌决策无声** —— 玩家读不到 AI 性格

### 5.3 实测清单（v0.1 → v0.2 之间必做）

需要在 v0.2 GDD 前完成的实测：

- [ ] 启动桌面端 → 录屏，标每个环节的实际时长 + 视觉反馈
- [ ] 启动到出第一手牌 → 测实际耗时（应 < 30 秒）
- [ ] 听完整 8 个音效文件 → 验证每个音效用对了动作
- [ ] 录屏 5 局完整对局 → 标每个决策的反馈延迟（应 < 200ms）
- [ ] 测高分屏（2x/3x）→ 验证视觉无模糊
- [ ] 测核心操作键盘快捷键 → 当前 0% 覆盖
