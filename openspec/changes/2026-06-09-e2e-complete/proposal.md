# 提案: E2E 测试补全

## 问题

e2e/ 目录有 37 个测试文件，但之前因 runtime 限制未验证全部通过。部分模块存在测试覆盖不足。

## 目标

确保所有 e2e 测试在运行时通过：
1. 验证 `dz_helpers.py` 提供所有测试所需的辅助函数
2. 确保所有 `debug_*` 和 `simulate_*` 方法存在且签名正确
3. 补齐缺失的测试（如有）

## 范围边界

**包含：**
- 验证所有 e2e 测试依赖的 `debug_`/`simulate_` 方法存在
- 修复任何缺失的方法
- 确保测试文件导入正确
- 运行 e2e 测试套件

**不包含：**
- 新测试用例（v0.7.0-M2 已在之前的变更中添加）
- 修改游戏逻辑

## 关键约束

1. **零行为变更** — 仅验证和修复测试基础设施
2. **不修改测试用例逻辑** — 只确保它们能运行
3. **所有测试必须通过** — 一个失败都不接受
