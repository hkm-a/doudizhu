# 设计: E2E 测试补全

## 策略

### 1. 依赖分析
- 解析所有 e2e 测试文件，提取所有 `root(game).call("...")` 调用
- 收集所有 `debug_*` 和 `simulate_*` 方法名
- 对比 `main.gd` 中实际存在的方法

### 2. 缺失方法修复
- 对缺失的方法在 `main.gd` 中添加
- 保持签名与 e2e 测试调用一致

### 3. 测试运行验证
- 使用 godot-e2e 框架启动游戏
- 逐个运行所有 e2e 测试
- 捕获失败日志

## 验收标准

- [ ] 所有 e2e 测试文件语法正确
- [ ] 所有测试依赖的 `debug_`/`simulate_` 方法存在
- [ ] `dz_helpers.py` 所有辅助函数存在
- [ ] e2e 测试套件至少能通过所有测试
